## 1. 現況分析

### 1.1 現有響應式支援

| 項目 | 現狀 | 問題嚴重度 |
|------|------|-----------|
| Layout 側邊欄 | 固定 `w-56`（224px），無法收合 | **嚴重** — 手機上佔滿螢幕寬度的一半以上 |
| 頁面內距 | 固定 `p-8`（32px） | **中等** — 手機上浪費大量空間 |
| 商品管理表格 | `<table>` 固定欄位 | **嚴重** — 手機上水平溢出或擠壓變形 |
| 文章管理 | 左右分欄 `w-80` + `flex-1` | **嚴重** — 手機上列表和詳情同時顯示太擠 |
| 商品探索 Grid | `grid-cols-2` 起步 | **輕微** — 已有基礎響應，但卡片內容在小螢幕略擠 |
| 儀表板 | `grid-cols-1 md:grid-cols-2` | **無** — 已是 mobile-first |
| 費用追蹤 | `grid-cols-1 md:grid-cols-3` | **無** — 已是 mobile-first |
| 工具列按鈕群 | `flex gap-3 flex-wrap` | **中等** — 有 flex-wrap 但按鈕太多時排列混亂 |
| Modal 彈窗 | `max-w-lg` + `p-4` | **輕微** — 基本可用 |
| 登入頁 | `max-w-md mx-auto` | **無** — 已置中適配 |

### 1.2 已有的響應式 class 使用

```
md:grid-cols-2  → DashboardPage（統計卡片）
md:grid-cols-3  → UsagePage（統計卡片）、ArticlesPage（Meta 資訊）
lg:grid-cols-4  → ExplorePage（商品卡片）
xl:grid-cols-5  → ExplorePage（商品卡片）
flex-wrap       → ProductsPage（工具列）
```

### 1.3 核心問題

1. **側邊欄無響應式處理** — 這是最大的阻礙，手機上 224px 固定側邊欄 + 內容區 = 不可用
2. **表格佈局** — `<table>` 在窄螢幕上無法自然適配
3. **雙欄佈局** — 文章管理的左側列表 + 右側詳情在手機上需要改為單欄切換

## 2. 設計原則

### 2.1 Mobile-First 策略

- 預設樣式為手機版，用 `md:`、`lg:` 向上擴展
- 現有桌面版體驗不可退化
- 觸控友善：按鈕最小 44x44px、間距足夠防誤觸

### 2.2 漸進式改造

- 不重寫元件，僅加入響應式 class
- 保持現有 Tailwind 4 inline class 模式
- 不引入新依賴（不需要 headless UI 或 hamburger 套件）

### 2.3 斷點定義

| 名稱 | Tailwind 斷點 | 寬度 | 目標裝置 |
|------|-------------|------|---------|
| 手機 | 預設（無前綴） | < 768px | iPhone / Android 手機 |
| 平板 | `md:` | ≥ 768px | iPad / Android 平板 |
| 桌面 | `lg:` | ≥ 1024px | 筆電 / 桌機 |
| 大桌面 | `xl:` | ≥ 1280px | 寬螢幕 |

## 3. 斷點策略

### 3.1 關鍵斷點行為

```
< 768px（手機）
├── 側邊欄：隱藏，改為底部導航列（Bottom Tab Bar）
├── 頁面內距：p-4（16px）
├── 商品管理：表格 → 卡片列表
├── 文章管理：單欄切換（列表 / 詳情）
├── 工具列：折疊為下拉式操作選單
└── 字體：標題 text-xl，正文 text-sm

≥ 768px（平板 / 桌面）
├── 側邊欄：正常顯示 w-56
├── 頁面內距：p-8（32px）
├── 商品管理：表格正常顯示
├── 文章管理：左右分欄
└── 工具列：正常水平排列
```

## 4. 元件改造清單

### 4.1 Layout.jsx — 導航系統（**最高優先級**）

**現狀：** 固定側邊欄 `w-56`，包含 logo、7+1 個導航項、用戶資訊、登出按鈕。

**改造方案：底部導航列（Bottom Tab Bar）**

```
桌面版（≥ 768px）：保持現有左側邊欄不變

手機版（< 768px）：
┌─────────────────────────────────┐
│                                 │
│         頁面內容區               │
│         (full width)            │
│                                 │
├─────────────────────────────────┤
│ 📊  🛒  🔍  📝  ⋯              │  ← 底部導航列（固定 Bottom Bar）
│ 儀表板 商品 探索 文章 更多        │
└─────────────────────────────────┘
```

**底部導航列設計：**
- 顯示 4 個主要頁面 icon + 「更多」按鈕
- 「更多」展開浮動選單：費用追蹤、設定、使用說明、（管理員：用戶管理）、用戶資訊、登出
- 固定在底部 `fixed bottom-0`，高度 ~56px
- 主內容區加 `pb-16` 避免被遮擋
- 使用 `md:hidden` / `hidden md:flex` 切換底部列 / 側邊欄

**技術要點：**
- 新增 state `moreMenuOpen` 控制「更多」彈出選單
- 點擊外部自動關閉（`useEffect` + click outside）
- 導航高亮使用現有 `NavLink` 的 `isActive`
- 用戶資訊、登出移入「更多」選單

### 4.2 頁面容器 — 內距調整

**現狀：** 各頁面頂層 `<div className="p-8">`

**改造：** `p-4 md:p-8`

影響頁面：DashboardPage、ProductsPage、SettingsPage、UsagePage、GuidePage、AdminPage

### 4.3 Toast 通知

**現狀：** `fixed top-6 right-6`

**改造：** `fixed top-4 right-4 left-4 md:left-auto md:right-6 md:top-6`

手機上變為頂部橫幅通知（full width），桌面上保持右上角。

## 5. 各頁面改造方案

### 5.1 DashboardPage（儀表板）

**現狀評估：** 已有 `grid-cols-1 md:grid-cols-2`，**改動最小**。

**改動：**
- 容器內距：`p-4 md:p-8`
- 公告區塊：無需調整（已是全寬流式佈局）
- Extension 狀態卡片：無需調整

**工作量：極小**

---

### 5.2 ProductsPage（商品管理）— **改動最大**

**問題：**
1. 工具列按鈕太多，手機上擠成多行
2. `<table>` 在手機上無法正常顯示 6 欄
3. 儲存連結面板的資訊密度太高
4. 排序拖拽區在手機上可用但體驗差

**改造方案：**

#### 工具列

```
桌面版：水平排列所有按鈕（現狀）

手機版：
┌─────────────────────────────────┐
│ 商品管理                    🔄  │  ← 標題 + 重整
├─────────────────────────────────┤
│ [🔗 匯入] [🚀 擷取(3)] [⋯ 更多]│  ← 常用按鈕 + 溢出選單
└─────────────────────────────────┘
```

- 「更多」按鈕收納：範本選擇、附圖選項、批量刪除
- 生成按鈕在選取商品時顯示為底部浮動按鈕（Floating Action Button）

#### 商品列表：表格 → 卡片

```
桌面版（≥ 768px）：保持 <table>

手機版（< 768px）：卡片列表
┌─────────────────────────────────┐
│ ☐ [商品圖] 商品名稱              │
│          $299  ⭐4.9  銷量 571   │
│          🔗 shopee.tw/xxx...     │
│          [📥 圖片] [🗑️ 刪除]     │
└─────────────────────────────────┘
```

**技術方案：**
- 手機版和桌面版用條件渲染（`<div className="hidden md:block">` for table, `<div className="md:hidden">` for cards）
- 不用 media query JS，純 CSS class 切換
- 卡片元件可從表格 row 的 JSX 抽取（資料結構相同）

#### 拖拽排序區

- 手機上改為垂直排列（`flex-col`）而非水平滾動
- 長按拖拽（touch 原生支援，`@dnd-kit` 已支援 touch）

**工作量：大**

---

### 5.3 ArticlesPage（文章管理）— **改動大**

**問題：**
1. 左側列表 `w-80`（320px）在手機上佔滿全寬
2. 右側詳情區需要滾動到底才看得完
3. 工具列 5 個按鈕在手機上過擠

**改造方案：單欄切換模式**

```
手機版行為：
1. 預設顯示文章列表（全寬）
2. 點擊文章 → 切換到詳情視圖（全寬）
3. 詳情頁頂部有「← 返回列表」按鈕

桌面版（≥ 768px）：保持現有左右分欄
```

**技術方案：**
- 新增一個 responsive state：在手機上 `selectedArticle` 控制顯示哪個面板
- 列表區：`block md:block md:w-80` → 手機全寬，桌面固定 320px
- 詳情區：`hidden md:block md:flex-1` → 手機隱藏，有選中文章時全螢幕顯示
- 手機版的「返回」按鈕：`md:hidden`

#### 工具列

```
手機版：
┌─────────────────────────────────┐
│ ← 返回            [狀態下拉]    │
│ 文章標題 ✏️                      │
├─────────────────────────────────┤
│ [📋複製] [📊SEO] [🚀優化] [⋯]  │  ← 按鈕換行 or 溢出選單
└─────────────────────────────────┘
```

- 「貼到 Dcard」和「刪除」收入「更多」溢出選單
- 按鈕文字在手機上可省略，只留 icon（`<span className="hidden md:inline">文字</span>`）

#### SEO 面板

- 已是可折疊設計，無需大改
- SVG 分數圓環在手機上縮小（`w-20 h-20 md:w-28 md:h-28`）

**工作量：大**

## 6. 技術實作路線

### 6.1 Phase 1：基礎架構（Layout + 全域樣式）

**目標：** 讓所有頁面在手機上至少「可用」

```
1. Layout.jsx 改造
   ├── 手機版底部導航列（Bottom Tab Bar）
   ├── 側邊欄 md: 以上才顯示
   ├── 主內容區加底部 padding
   └── 「更多」選單（用戶資訊 + 次要頁面 + 登出）

2. 全域內距調整
   └── 所有頁面 p-8 → p-4 md:p-8

3. Toast 通知適配
   └── 手機全寬 banner
```

**預估改動檔案：** 2 個（Layout.jsx + 各 Page 的 p-8 → p-4 md:p-8）

### 6.2 Phase 2：重點頁面改造

**目標：** 高使用頻率頁面在手機上「好用」

```
1. ArticlesPage.jsx
   ├── 單欄切換（列表 ↔ 詳情）
   ├── 返回按鈕
   └── 工具列響應式

2. ProductsPage.jsx
   ├── 表格 → 卡片（手機版）
   ├── 工具列溢出選單
   └── 拖拽排序垂直化
```

**預估改動檔案：** 2 個

### 6.3 Phase 3：次要頁面 + 細節打磨

```
1. ExplorePage.jsx
   ├── Grid 調整（手機 1 欄）
   ├── Tab 水平滾動
   └── 篩選面板折疊

2. SettingsPage.jsx
   └── 範本編輯區域適配

3. AdminPage.jsx
   └── 表格卡片化

4. UsagePage.jsx
   └── 趨勢圖容器調整
```

**預估改動檔案：** 4 個

## 8. 測試策略

### 8.1 測試斷點

| 裝置 | 寬度 | 必須測試 |
|------|------|---------|
| iPhone SE | 375px | 最小可用寬度 |
| iPhone 14 | 390px | 主流 iOS |
| Android 中階 | 360px | 主流 Android |
| iPad Mini | 768px | 平板臨界值（md: 斷點） |
| iPad | 1024px | lg: 斷點 |

### 8.2 測試重點

- [ ] 底部導航列在所有手機寬度正常顯示
- [ ] 「更多」選單正確展開/收起
- [ ] 導航切頁時高亮正確
- [ ] ProductsPage 卡片列表正常渲染
- [ ] ArticlesPage 列表→詳情→返回 流程順暢
- [ ] 商品拖拽排序在觸控裝置可用
- [ ] 所有 Modal 在手機上不溢出
- [ ] 所有按鈕觸控面積 ≥ 44x44px
- [ ] 橫屏（landscape）不破版
- [ ] 輸入框在手機鍵盤彈出時不被遮擋

### 8.3 測試工具

- Chrome DevTools Device Toolbar（開發時快速驗證）
- 實機測試（iOS Safari + Android Chrome，最終驗收）
- `npm run build` + 預覽確認（production 行為一致）

## 附錄：不需要新增的依賴

本方案完全基於現有技術棧：

- **Tailwind CSS 4** 的響應式 class（`md:`, `lg:`, `xl:`）
- **React state** 控制手機版 UI 切換
- **@dnd-kit** 已原生支援 touch 事件

不需要引入：`@headlessui/react`、`react-responsive`、hamburger menu 套件、或其他 UI 框架。
